# Project Brief

## 项目名称
暂定：未命名 2D 肉鸽原型

## 项目类型
- 引擎：Godot 4.x
- 类型：2D Roguelike / Survivors-like
- 视角：俯视角
- 开发目标：快速做出一个可反复试玩的战斗成长原型

## 项目目标
用较小开发成本，完成一个具备以下特点的原型：
- 上手快
- 战斗节奏清晰
- 成长反馈明显
- 有初步 build 构筑空间
- 能支持快速版本迭代

## 核心体验关键词
- 生存压力
- 高频成长
- 走位规避
- 随机选择
- 构筑成型

## 首期范围
### 必须包含
- 玩家移动
- 基础攻击机制
- 至少 3 类敌人
- 敌人刷新与强度提升
- 经验掉落与升级三选一
- 死亡与重开
- 基础 UI

### 可选增强
- 精英怪
- Boss
- 特殊事件房或补给点
- 局外成长
- 多角色差异

## 核心循环
1. 开局进入战斗
2. 击败敌人并拾取经验
3. 升级后从多个成长项中选择其一
4. 随时间面对更强敌人和更密集压力
5. 依靠 build 与操作活得更久
6. 死亡结算并开始下一轮迭代

## 版本目标
### v0.1 目标
打通闭环，确保能完整玩一局

### v0.2 目标
增加内容差异，提升 build 体验

### v0.3 目标
增加中后期目标，例如精英、事件或 Boss

## 技术目标
- 工程结构清晰
- 数据驱动便于加内容
- 核心逻辑低耦合
- 支持高频修改参数与内容

## 成功指标
- 玩家能在 1 分钟内理解操作和目标
- 单局可稳定运行 10 分钟
- 每次升级都能明显感受到成长
- 不同 build 至少能形成 2 种以上可识别倾向
- 每轮试玩都能产生明确可执行的改进项

## 风险点
- 前期内容太少导致体验单薄
- 数值膨胀过快导致后期失控
- 升级项同质化导致选择无意义
- 程序写死过多导致后续扩展困难

## 协作要求
- 策划输出规则、目标、验收标准
- 程序输出实现、文件变更、风险说明
- 试玩输出 bug、体验、优先级、复现步骤
